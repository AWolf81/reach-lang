1.5.1 JavaScript

The Reach JavaScript backend produces a compilation output named "input.mjs" which exports an asynchronous function for each participant. Each function accepts 3+n arguments where the first three arguments are stdlib, ctc, and interact, while the remaining arguments are the initial local state of the participant. These functions should be called by the frontend.

The stdlib argument is provided by either

  • the module @reach-sh/stdlib/ETH.mjs; or,

  • the modules @reach-sh/stdlib/ALGO.mjs.

The ctc argument is the result of a call to acc.deploy or acc.attach.

The interact argument is an object that has a method for each interaction in the corresponding participant’s local computation.

The stdlib modules export the following functions that might be used in this frontend.

—
newTestAccount(balance) => acc

Returns a Reach account abstraction for a new account on the consensus network with a given balance of network tokens. This can only be used in private testing scenarios, as it uses a private faucet to issue network tokens.

—
connectAccount(networkAccount) => acc

Returns a Reach account abstraction for an existing account for the consensus network based on the connector-specific account specification provided by the networkAccount argument.

—
acc.address => networkAccount

Returns the connector-specific account specification of a Reach account abstraction.

—
acc.deploy(bin) => ctc

Returns a Reach contract abstraction after deploying a Reach DApp contract based on the bin argument provided. This bin argument is the "input.mjs" module produced by the JavaScript backend.

—
acc.attach(bin, ctc) => ctc

Returns a Reach contract abstraction based on a deployed Reach DApp contract provided in the ctc argument and the bin argument. This bin argument is the "input.mjs" module produced by the JavaScript backend.

—
balanceOf(acc) => amount

Returns the balance of network tokens held by the account given by a Reach account abstraction provided by the acc argument.

—
transfer(from:networkAccount, to:networkAccount, amount) => void

Transfers amount network tokens from from to to, which are provided by connector-specific account specifications.

—
checkType(t, x) => x

Asserts that value x has reach type t. An exception is thrown if this is not the case. See the table below for Reach types and their corresponding JavaScript representation:

// Reach => JavaScript
'bool'    => 'boolean'
'uint256' => 'BN'
'bytes'   => 'string'
'address' => 'string'

—
toBN(x) => uint256
isBN(x) => bool

toBN converts a JavaScript number to a BN, the JavaScript representation of Reach’s uint256. isBN checks if its input is a BN.

—
toHex(x) => bytes
isHex(x) => bool
hexToBN(bytes) => uint256
bnToHex(uint256) => bytes
uint256_to_bytes(uint256) => bytes
bytes_eq(bytes, bytes) => bool

These are additional conversion and comparison utilities.

—
keccak256(x) => uint256

Hashes the value.

—
random_uint256() => uint256

Creates 256 random bits as a uint256.

—
add(uint256, uint256) => uint256
sub(uint256, uint256) => uint256
mod(uint256, uint256) => uint256
mul(uint256, uint256) => uint256
div(uint256, uint256) => uint256

Integer arithmetic on uint256.

—
eq(uint256, uint256) => bool
ge(uint256, uint256) => bool
gt(uint256, uint256) => bool
le(uint256, uint256) => bool
lt(uint256, uint256) => bool

Integer comparisons on uint256.

—
toWei(ether) => wei
fromWei(wei) => ether
toWeiBN(ether) => uint256

Wei conversion functions only exported by the stdlib for ETH.
