1.4.1 Ethereum

The Ethereum Reach connector generates a contract that manages one instance of the DApp’s execution. It is guaranteed to use exactly one word of on-chain state.

The connector provides a binding named ETH to backends.

During compilation, the connector produces two intermediate outputs corresponding to each of these fields: "input.sol", containing the Solidity code implementing the contract, as well as "input.evm", containing experimental, internally generated EVM assembly, which is not used or supported.
